67  // A partition is an object containing data about a set of pattern vertices
68  // and their domains.
69  // Each partition should have vertices and domains
70  // disjoint from those of every other partition.
71  //
72  // (Thus, we can search for Hall sets, and reduce domains,
73  // within each partition, independently of all others; they cannot
74  // affect each other, as far as Hall set reduction is concerned).
75  //
76  // Note that, even if a Hall set is detected, it still may be worth
77  // storing the domains and checking later for further reductions.
78  // E.g., if initially we have Dom(x1) = Dom(x2) = Dom(x3) = {y1, y2, y3},
79  // then y1,y2,y3 can be removed from all other domains.
80  // But then we move x1,x2,x3 into their own new partition.
81  // If in future some other reduction,
82  // unrelated to Hall sets, reduces it to Dom(x1) = Dom(x2) = {y1,y2},
83  // then Hall set reduction will detect this and force x3->y3 (or a nogood,
84  // if y3 was removed from Dom(x3) by some other reduction).

86  std::vector<Data> domains_data;
