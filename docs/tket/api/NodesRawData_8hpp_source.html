24 // The "logical" data structure for a search tree would be a std::vector
25 // of "node" objects, with each node containing an object like
26 // std::map<VertexWSM, std::set<VertexWSM>> (with key being the PV,
27 // and value being Domain(pv)).
28 //
29 // Think of this as a 2D structure,
30 // with x-axis being the PV, and y-axis being the node index
31 // (i.e., the index in the std::vector of "node" objects).
32 //
33 // (Thus, logically, we have a single vertical strand - the nodes vector -
34 // with each node having a horizontal bar - the domains).
35 //
36 // However, for better performance we reduce copying
37 // by only storing Domain(pv) when it CHANGES.
38 //
39 // Thus, the actual data structure is quite different; each PV
40 // has its own std::vector of DOMAINS. Thus, we have a whole collection
41 // of "vertical strands" (columns) - one for each PV.
42 //
43 // We need two kinds of movement along the data: "horizontal"
44 // (i.e., for the current logical node, get information about the domains),
45 // provided by the class DomainsAccessor;
46 // and "vertical" (i.e., as we search, we move up and down the std::vector
47 // of node objects), provided by the class NodeListTraversal.
48 //
49 // Thus, this raw data is SHARED between these two classes, each of which
50 // understands how to do its own kind of motion.
