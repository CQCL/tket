76  protected:
77  /*
78  Most natural reducers have a symmetry. Suppose that
79 
80  pv1 -> tv1 gives a map (function)
81  M(reducer,pv1,tv1) : {all PV} -> PowerSet({all TV}),
82 
83  meaning (by definition) that: for every pv2,
84 
85  Domain(pv2) is a subset of S = M(reducer,pv1,tv1)[pv2]
86 
87  (i.e., as soon as we know that the assignment pv1 -> tv1
88  has been made, we can intersect the current Domain(pv2) with S).
89 
90  [We can of course let S be the set of all TV for some pv2,
91  in which case this reducer has no effect upon Dom(pv2),
92  i.e. pv1, pv2 do not affect each other, as far as this reducer knows].
93 
94  THEN, it is often the case that if subsequently we reduce further
95  (in any way - perhaps from other unrelated reducers),
96  and obtain a new assignment pv2 -> tv2,
97  (so that tv2 is in M(reducer,pv1,tv1)[pv2]),
98  then our reducer is automatically consistent with pv1;
99  we do not need to check if pv1 -> tv1 is valid (for this reducer).
100 
101  Another way to phrase this: the reducer satisfies
102 
103  tv2 in M(reducer,pv1,tv1)[pv2] <==>
104  tv1 in M(reducer,pv2,tv2)[pv1].
105 
106  E.g., if "reducer" is the distance reducer with parameter d, then:
107 
108  M(., pv1, tv1) maps every pv2 with dist(pv1, pv2) = d
109  to the set { tv2 : dist(tv1, tv2) <= d }.
110 
111  Thus by definition:
112 
113  tv2 in M(.,pv1,tv1)[pv2] <==> dist(pv1,pv2)=d, dist(tv1,tv2)<=d.
114 
115  It's obvious from this that we can swap pv1,pv2 and tv1,tv2.
116  */
