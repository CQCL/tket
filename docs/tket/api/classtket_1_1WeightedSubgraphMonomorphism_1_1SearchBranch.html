Detailed Description

Represents a depth-first traversal.

However, it's really about reducing nodes etc., not the search logic. This contains the list of nodes so far, which may be viewed as a stack, but the caller is responsible for moving up and down the list.

◆ get_used_target_vertices()

std::set< VertexWSM > tket::WeightedSubgraphMonomorphism::SearchBranch::get_used_target_vertices ( ) const

A relatively expensive operation.

Simply returns ALL target vertices which still exist in SOME domain, ANYWHERE in the search tree. Thus, anything NOT in this set can be safely erased (as far as THIS search is concerned, anyway). Naturally, as the search progresses this set will reduce, so for best results the caller should delay this as long as possible.

Returns
The set of all target vertices which might still occur during the future search.
