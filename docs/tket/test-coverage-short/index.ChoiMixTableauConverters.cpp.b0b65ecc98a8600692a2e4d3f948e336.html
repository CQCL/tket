41 * THE PLAN:
42 * We first identify and solve all post-selections by Gaussian elimination and
43 * diagonalisation of the input-only rows. This provides us with better
44 * guarantees about the form of the stabilizers generated by the remaining
45 * tableau - specifically that every possible stabilizer will involve at least
46 * some output portion, so out_qb will always be set in later sections of the
47 * synthesis. Diagonalisation of the input-only rows reduces them to just Z
48 * strings but not necessarily over a minimal number of qubits. This can be
49 * achieved by taking the Z matrix of these rows and performing row-wise
50 * Gaussian elimination. The leading 1s indicate which qubits we are isolating
51 * them onto and the rest gives the CXs required to reduce them to just the
52 * leading qubits. After all post-selections have been identified, we enter
53 * the main loop of handling all other inputs and reducing them one at a time
54 * to either an identity wire or Z-decoherence (OpType::Collapse) connected to
55 * an output, or to a discard. Let in_qb be this qubit to be solved. Pick a
56 * row containing X_in_qb (if one exists). Since it must contain some
57 * non-identity component in the output segment, we can pick one of these to
58 * be out_qb and apply unitary gates at the input and output to reduce the row
59 * to X_in_qb X_out_qb. We do the same with Z (if a row exists) to necessarily
60 * give Z_in_qb Z_out_qb by commutativity properties, or we pick out_qb here
61 * if no X row was found. If we had both an X row and a Z row, we have reduced
62 * it to an identity wire just fine. If we have only one of them, e.g. X_in_qb
63 * X_out_qb, we wish to make it the only row with X on either in_qb or out_qb
64 * to leave a decoherence channel. We note that any other row containing
65 * X_out_qb must have some other P_out2, since their combination cannot leave
66 * an empty output segment. So we can apply a unitary gate to eliminate the
67 * X_out_qb on the other row. By doing output-first gaussian elimination on
68 * the output sub-tableau ignoring the target row and X_out_qb column, for
69 * each such row with X_out_qb there is some other output column P_out2 for
70 * which it is the unique entry, so we can be sure that applying the unitary
71 * gate does not add X_out_qb onto other rows. Having this strategy available
72 * to make it the unique row with X_out_qb still allows us to make it the
73 * unique row with X_in_qb by row combinations. Once all rows with inputs have
74 * been eliminated, any unsolved inputs must have been discarded and the
