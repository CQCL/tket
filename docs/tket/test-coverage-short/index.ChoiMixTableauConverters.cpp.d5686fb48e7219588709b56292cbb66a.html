44 /**
45 * We will consider applying gates to either side of the tableau to reduce
46 * qubits down to one of a few simple states (identity, collapse, zero
47 * initialise, mixed initialised, post-selected, or discarded), allowing us to
48 * remove that qubit and continue until the tableau contains no qubits left.
49 * This gradually builds up a set of operations both before and after the
50 * working tableau. We should ask for the following combination of temporary
51 * states to compose to form a channel equivalent to that described by the
52 * input tableau:
53 * - in_circ: a unitary circuit (without implicit permutations)
54 * - post_selected: a set of post-selection actions to apply to the outputs of
55 * in_circ
56 * - discarded: a set of discard actions to apply to the outputs of in_circ
57 * - collapsed: a set of collapse actions to apply to the outputs of in_circ
58 * (i.e. decoherence in the Z basis)
59 * - tab: the remaining tableau still to be solved. Acting as an identity on
60 * any qubits not contained within tab
61 * - in_out_permutation: a permutation of the qubits, read as a map from the
62 * input qubit name to the output qubit it is sent to
63 * - zero_initialised: a set of initialisations of fresh output qubits
64 * (in_out_permutation may join these qubits onto input qubits that have been
65 * post-selected or discarded to reuse qubits)
66 * - mix_initialised: a set of initialisations of output qubits into
67 * maximally-mixed states (in_out_permutation may similarly join these onto
68 * input qubits no longer in use)
69 * - out_circ_tp: the transpose of a unitary circuit (without implicit
