26 /** A general interface for an object which knows, using various
27 * graph-theoretic tools, that if a new assignment PV->TV is made,
28 * then certain other domains can be reduced,
29 * thus possibly pruning the search space and speeding up the search.
30 * However, these objects do NOT remember which assignments were previously
31 * processed.
32 *
33 * The reason for having a "check" and "reduce" function is that, in most
34 * natural cases, the SAME graph theory which enables a particular reduction
35 * ALSO can be applied to give a useful "check" function.
36 */
37 class ReducerInterface {
38 public:
39 /** Check if pv->tv may be valid, considered in isolation from all other
40 * assignments. This should be cheaper than a reduction. By default, just
41 * returns true always.
42 * Of course it is a waste of time to call this multiple times with the same
43 * assignment; the CALLER is responsible for arranging things so as to avoid
44 * duplicated calls.
45 * Note that this doesn't actually alter any domains (it doesn't even know
46 * about the current domains). The caller should erase PV->TV completely
47 * from ALL data whenever this returns false.
48 * @param assignment An assignment PV->TV under consideration.
49 * @return False if the assignment is and was ALWAYS invalid (NOT just because
50 * of the other domains at this exact moment in the search). Thus, TV can be
51 * erased from EVERY Domain(PV) in EVERY node of EVERY search, not just the
52 * current node).
53 */
54 virtual bool check(std::pair<VertexWSM, VertexWSM> assignment) = 0;
55
56 /** Given that PV->TV is a new assignment, reduces the domains
57 * of all affected vertices. Breaks off early
58 * if new assignments arise in the node due to reductions.
59 * Of course, UNLIKE when the "check" function returns false, a nogood here
60 * does NOT mean that PV->TV is invalid always;
61 * just that it is invalid IN COMBINATION with the complete collection
62 * of domains in the CURRENT node.
63 * @param assignment An assignment PV->TV under consideration.
64 * @param accessor An object providing read/write access to the complete
65 * collection of domains in the CURRENT node.
66 * @param work_set A reusable object which may or may not be used by this
67 * class, to avoid unnecessary memory reallocation.
68 * @return After reducing the domains, information about what happened; did we
69 * hit a nogood? Did we create a new assignment?
70 */
71 virtual ReductionResult reduce(
72 std::pair<VertexWSM, VertexWSM> assignment, DomainsAccessor& accessor,
73 boost::dynamic_bitset<>& work_set) = 0;

77 protected:
78 /*
79 Most natural reducers have a symmetry. Suppose that
80
81 pv1 -> tv1 gives a map (function)
82 M(reducer,pv1,tv1) : {all PV} -> PowerSet({all TV}),
83
84 meaning (by definition) that: for every pv2,
85
86 Domain(pv2) is a subset of S = M(reducer,pv1,tv1)[pv2]
87
88 (i.e., as soon as we know that the assignment pv1 -> tv1
89 has been made, we can intersect the current Domain(pv2) with S).
90
91 [We can of course let S be the set of all TV for some pv2,
92 in which case this reducer has no effect upon Dom(pv2),
93 i.e. pv1, pv2 do not affect each other, as far as this reducer knows].
94
95 THEN, it is often the case that if subsequently we reduce further
96 (in any way - perhaps from other unrelated reducers),
97 and obtain a new assignment pv2 -> tv2,
98 (so that tv2 is in M(reducer,pv1,tv1)[pv2]),
99 then our reducer is automatically consistent with pv1;
100 we do not need to check if pv1 -> tv1 is valid (for this reducer).
101
102 Another way to phrase this: the reducer satisfies
103
104 tv2 in M(reducer,pv1,tv1)[pv2] <==>
105 tv1 in M(reducer,pv2,tv2)[pv1].
106
107 E.g., if "reducer" is the distance reducer with parameter d, then:
108
109 M(., pv1, tv1) maps every pv2 with dist(pv1, pv2) = d
110 to the set { tv2 : dist(tv1, tv2) <= d }.
111
112 Thus by definition:
113
114 tv2 in M(.,pv1,tv1)[pv2] <==> dist(pv1,pv2)=d, dist(tv1,tv2)<=d.
115
116 It's obvious from this that we can swap pv1,pv2 and tv1,tv2.
117 */

137 * have already been processed.

139 class ReducerWrapper {

143 /** Call at the start, when we are about to begin reducing a node. */
144 void clear();

146 /** Checks if the given PV->TV assignment appears to be valid,
147 * separately from others (i.e., in isolation).
148 * Does NOT keep track of whether it was checked before.
149 * For convenience, just wrap around the "check" function of the
150 * ReducerInterface object.
151 * @param assignment An assignment PV->TV under consideration.
152 * @return False if the assignment is and was ALWAYS invalid, in ALL search
153 * nodes.
154 */
155 bool check(std::pair<VertexWSM, VertexWSM> assignment);
156
157 /** Starts to reduce domains using any new assignments which arose since
158 * the last call.
159 * Keeps track of previously processed assignments and doesn't repeat them.
160 * Breaks off early if a new assignment is created, but will subsequently
161 * resume automatically from the next unprocessed assignment if this occurs.
162 * The reason is that new assignments have cascading effects, often reducing
163 * many other domains simultaneously and cheaply. Thus it is better to let
164 * the caller perform these cheap reductions first, before carrying out the
165 * reductions in this object (which are usually much more expensive).
166 * @param accessor An object to get information about the current domains, and
167 * alter them if necessary. Also, gets the list of new assignments from this
168 * object.
169 * @param work_set An object for reuse, to avoid expensive memory
170 * reallocations.
171 * @return The result of reducing the current node with ALL new assignments
172 * not yet processed. Can be resumed later if breaking off early due to new
173 * assignments occurring.
174 */
175 ReductionResult reduce(
176 DomainsAccessor& accessor, boost::dynamic_bitset<>& work_set);
177
178 private:
179 ReducerInterface& m_reducer;
180 std::size_t m_number_of_processed_assignments;
181 };
